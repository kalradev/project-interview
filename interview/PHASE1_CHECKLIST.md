# Phase 1 – Company X flow (how to verify)

Use this to check that the project works the way you want for **Company X**.

---

## 1. Resumes on admin platform

- **From platform:** Admin dashboard → **"From platform"** → paste resume text (must contain an email) + job role → **Submit from platform**.
- **Agent** computes ATS score from the resume. If **ATS ≥ 85%**, the user is **shortlisted**: candidate is created and an **invite email** is sent (SMTP/Brevo). If ATS < 85, no candidate is created; the response shows the score and "Not shortlisted".
- **Manual add:** **"+ Add resume"** → fill form and use the **"Send invite & schedule interview"** toggle to send email or not.

---

## 2. User list on admin dashboard with details and photos

- **Candidates** list shows: **Photo** (thumbnail after interview), **Name**, **Email**, **Job role**, **ATS**, **Status**, **Invited** date, **Actions**.
- After the interview, the **before-interview photo** appears in the **Photo** column (and in the report).
- Filter by **Status**: All, Invited, Selected, etc.

---

## 3. Interview link and setup via SMTP; email from resume

- Shortlisted users receive an **email** (SMTP if configured in `.env`, else Brevo) with:
  - **Email** = address **extracted from the resume**
  - **Password** = generated by the system
  - **Interview timing**
  - **Setup link** = `SETUP_APP_DOWNLOAD_URL` (Interview Agent download).
- Configure in `interview/.env`: `SMTP_*` or `BREVO_*` and `SETUP_APP_DOWNLOAD_URL`.

---

## 4. Interview by role + tech stack + communication

- The **AI interviewer** asks questions based on:
  - **Job role** the user applied for
  - **Tech stack** from their profile (from platform or manual add)
  - **Communication skills** (and related soft skills)
- The desktop app sends **tech_stack** when fetching the next question so the agent can ask role- and tech-relevant questions.

---

## 5. Report to admin with result and accuracy

- When the candidate **ends the interview**, the backend generates an **agent report**:
  - **Accuracy score** (0–100) – how correct/technical the answers were
  - **Communication score** (0–100)
  - **Summary** (for admin)
  - **Interrupt count** – times the user “cut” the agent (submitted too quickly, e.g. &lt; 15 s)
- Admin opens **Report** for a candidate to see: **Agent result**, **Q&A**, **photos**, **video**, **integrity score**, **face/lip check**.

---

## 6. Agent stops / interrupt detection (user cutting agent)

- If the user submits an answer **within 15 seconds** of the question being shown, it is recorded as **answered_quickly** (user “cut” the agent).
- **Interrupt count** is stored and shown in the **agent report** on the admin dashboard.
- (Stopping the interview automatically after N interrupts can be added later; for now the count is reported.)

---

## 7. Face expression and lip-sync (cheating check)

- **Before interview:** One photo (profile photo).
- **During interview:** Periodic **session photos** (e.g. every 2 minutes) and **video** recording.
- **Face/lip status** is set when the session ends: **pass** (photos with face), **review**, or **pending** (no session photos).
- Admin **Report** shows **Face & lip check** and all **photos + video** for manual verification.

---

## 8. Full report on admin dashboard

The **Report** modal for a candidate includes:

- **Agent result:** Accuracy %, Communication %, Summary, Times user cut agent
- **Face & lip check:** pass / review / pending
- **Integrity score & risk**
- **Before-interview photo** and **during-interview photos**
- **Interview video**
- **Q&A** (all exchanges)

---

## Quick test order

1. **Backend:** `cd interview && .\run_server.ps1` (and run `python scripts/add_ats_and_photos.py` once if you haven’t).
2. **Admin dashboard:** `cd admin-dashboard && npm run dev` → login as admin → **From platform** → paste a resume that contains an email and enough text to get ATS ≥ 85 → submit → check that the candidate appears and an email is sent.
3. **Desktop app:** Candidate installs/runs the app → logs in with email from the invite and password → **Photo** (before) → **Interview** (video on, questions by role + tech + communication) → answer a few questions (try one “quick” submit &lt; 15 s to see interrupt count) → **End interview**.
4. **Admin dashboard:** Open **Report** for that candidate → confirm **Agent result**, **Face & lip check**, **photos**, **video**, and **Q&A**.
